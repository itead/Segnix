7 # What Is It?
8 
9 Segnix is an open-source interface library for hardware manipulation
10 based on Linux platform and compatible with [Arduino API][1].
11 You can develop with it by using C/C++ and Python. Segnix is dedicated to provide
12 a fast, efficient and unified software development environment for the person
13 like you with a strong electronic design idea.
14 
15 # CAUTION!
16 
17 ## Before November 22, 2014, Segnix was named ITEAD-OS-SDK
18 
19 # What Can It Do?
20 
21 Through Segnix, users can directly access to the Linux platform hardware
22 resources such as GPIO, UART, SPI, I2C etc. Segnix supports various
23 hardware platforms, such as Iteaduino Plus A10/A20, IBOX,
24 ITEAD Core EVB, Raspberry Pi and Raspberry Pi Model B+ and so on. Meanwhile, It also
25 provides a great amount of libraries for all kinds of child board like NFC(PN532)
26 module, GPS shield, GSM(SIM900/SIM908) shield, nRF24L01 module, OLED and LCD module etc.
27 
28 Simplifying the hardware operation makes it possible for software engineers
29 who are not familiar with the underlying hardware and junior developers to
30 understand more about "Code World" and "Physical World". It can also work
31 as a rapid prototyping development tool.
32 
33 # Features
34 
35 At present, Segnix has a set of **core features** as below:
36 
37  - Support for external interrupts operations
38  - Support for upper-layer time functions
39  - Support for PWM output with any GPIO
40  - Support for GPIO operations
41  - Support for GPIO 8bit/16bit bus operations
42  - Support for UART bus operations
43  - Support for I2C bus operations
44  - Support for SPI bus operations
45 
46 Based on core features, plenty of **functional features** can be reached:
47 
48  - Support for SSD1306-based I2C OLED screen operations
49  - Support for 1602 LiquidCrystal module
50  - Support for GPS module
51  - Support for GSM(SIM900/SIM908) module including GPRS/GPS operations
52  - Support for NFC(PN532) module
53  - Support for nRF24L01 module
54  - Support for Temperature and Humidity module

187 # How to Start
188 
189 [Segnix Python API][2] will be recommanded for Python users.
190 Segnix works well with C(.c) and C++(.cpp) programs. Especially, it is
191 compatiable with Arduino Sketch(.ino).
192 
193 ## To use Segnix in Python
194 
195 Create a new file named `blink.py` which reads as follow:
196 
197  #! /usr/bin/env python
198  from iteadsdk import *
199  LED = PG9
200  pinMode(LED, OUTPUT)
201  while True:
202  digitalWrite(LED, HIGH)
203  delay(1000)
204  digitalWrite(LED, LOW)
205  delay(1000)
206 
207 Make `blink.py` executable:
208 
209  $ chmod +x blink.py
210 
211 Then, run it:
212 
213  $ sudo ./blink.py
214 
215 You can get commutative HIGH and LOW from GPIO PG9, if you've run it on ITEAD CORE EVB
216 or other platforms based on Allwinner A10/A20 SoC.
217 
218 ## To use Segnix in C program
219 
220 Create a new file named "led.c" which reads as follow:
221 
222  #include <itead.h>
223  #include <stdio.h>
224 
225  #define STATUS_LED1 PG9
226 
227  void setup(void)
228  {
229  printf("setup begin\n");
230  pinMode(STATUS_LED1,OUTPUT);
231  }
232  void loop(void)
233  {
234  digitalWrite(STATUS_LED1,HIGH);
235  delay(1000);
236  digitalWrite(STATUS_LED1,LOW);
237  delay(1000);
238  }
239  void main(int argc, char **argv)
240  {
241  setup();
242  while(1)
243  {
244  loop();
245  }
246  }
247 
248 Then run iteadcompile in the terminal
249 
250  $ iteadcompile LED led.c
251 
252 You can run the LED program after compilation:
253 
254  $ sudo ./LED

326 #### Note: How to understand debug information from compiler
327 
328 You DO NOT include the itead.h explicitly in your source file with suffix .ino.
329 iteadcompile will take care of everything. That means you need to know the method
330 you should use for locating your .ino sketch bugs. Actually,if you have a sketch
331 named "Hello.ino" with a bug at level ERROR in line 25, you will get some ERROR
332 informations telling the ERROR bug in "Hello.cpp" with line 26(25+1) when
333 command "iteadcompile Hello Hello.ino" is executed. Why? iteadcompile insert one
334 line like `include "itead.h"` in the head of your Hello.ino, change the suffix
335 .ino to .cpp and compile(using g++) Hello.cpp to generate Hello which is
336 executable binary file. When you encounter other bugs in your sketch,debug
337 skills on C++ program will also helpful for solving your problems.
