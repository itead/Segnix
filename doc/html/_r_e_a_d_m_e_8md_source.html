4 # ITEAD-SDK
5 
6 # What Is It?
7 
8 ITEAD-SDK is an open-source interface library for hardware manipulation
9 based on Linux platform and compatible with Arduino API. You can work
10 with it by using C/C++ and Python. ITEAD-SDK is dedicated to provide
11 a fast, efficient and unified software development environment for the
12 person like you with a strong electronic design idea.
13 
14 # Which Language?
15 
16 You can implement your design based on ITEAD-SDK in **C** or **C++**.
17 
18 **Especially and amazingly**, **ITEAD-SDK Python API** is provided for Python users.
19 **A few beautiful lines of code will achieve your goals!**
20 
21 # What Can It Do?
22 
23 Through ITEAD-SDK, users can directly access to the Linux platform hardware
24 resources such as GPIO, UART, SPI, I2C etc. ITEAD-SDK supports various
25 hardware platforms. For now, it supports Iteaduino Plus A10/A20 Core, ibox
26 and Raspberry Pi. Meanwhile, It also provides a great amount of libraries
27 for all kinds of child board like NFC(PN532) module, GPS shield,
28 GSM(SIM900/SIM908) shield, nRF24L01 module, OLED and LCD module etc. Without
29 any codes modified, the libraries and users codes for Arduino can be run on
30 ITEAD-SDK. ITEAD-SDK will support much more hardware platforms and
31 functional modules in the future. In the Linux thriving time, ITEAD-SDK
32 will make any impossible fanatic idea possible, are you ready?
33 
34 Simplifying the hardware operation makes it possible for software engineers
35 who are not familiar with the underlying hardware and junior developers to
36 understand more about "Code World" and "Physical World". It can also work
37 as a rapid prototyping development tool.
38 
39 The SDK's API, which is similar to Arduino, make it easier for those who have
40 some understanding of Arduino. Moreover, with the SDK, Python users can operate
41 the underlying hardware more easily.
42 
43 
44 Additionally, ITEAD-SDK provides most of Arduino API. Your sketches run
45 perfectly with or without few changes.
46

80 # Installation and Uninstallation
81 
82 Before all, we assume that you have installed the required packages:
83 `make build-essential git-core python python-dev` in your Debian system. If not, you can install
84 them by command:
85 
86  $ sudo apt-get update

93 ## Download and Install SDK in Debian
94 
95 Only a few steps, DO IT !
96 
97 ### Step 1: Getting the Source Code
98 
99 You can download SDK by command:
100 
101  $ git clone https://github.com/itead/SDK.git
102  $ cd SDK
103 
104 Or you can get SDK via other ways you prefer at <https://github.com/itead/SDK>
105 
106 #### NOTE
107 
108 All the commands of the following steps should be run under SDK's
109 source code directory.
110 
111 ### Step 2: Configure Your Board
112 
113 For now, platforms supported by SDK and the configuration commands are below.
114 You **MUST** choose the one fitting your board.
115 
116 - Iteaduino Plus A10/A20
117 
118  $ make Iteaduino_Plus
119 
120 - ITEAD CORE EVB
121 
122  $ make ITEAD_CORE_EVB
123 
124 - Raspberry Pi Rv2
125 
126  $ make Raspberry_Pi_Rv2
127 
128 - Raspberry Pi Model B+
129 
130  $ make Raspberry_Pi_Model_BPlus
131 
132 ### Step 3: Select Functional Libraries to Install (Optional)
133 
134 **If you want to install all the libraries, just skip this step.**
135 
136 It's allowed for user to select functional libraries to install. Open file
137 `config.mk` and edit it by **adding or removing letter "#"** at the head of each
138 line for **deselecting or selecting** the corresponding library. More
139 description for selecting libraries can be found in `config.mk`. For example:
140 
141  $ vi config.mk
142  Editing... and Save

145 
146 Installing SDK requires root(super user)'s privilege. Run command:

148  $ make

153  $ sudo make SDK_Python_install

177 # How to Use
178 
179 **ITEAD-SDK Python API will be recommanded for Python users.**
180 **SDK works well with C(.c) and C++(.cpp) programs. Especially, it is
181 compatiable with Arduino Sketch(.ino).**
182 
183 ## To use SDK in Python
184 
185 Create a new file named `blink.py` which reads as follow:
186 
187  #! /usr/bin/env python
188  from iteadsdk import *
189  LED = PG9
190  pinMode(LED, OUTPUT)
191  while True:
192  digitalWrite(LED, HIGH)
193  delay(1000)
194  digitalWrite(LED, LOW)
195  delay(1000)
196 
197 Make `blink.py` executable:
198 
199  $ chmod +x blink.py
200 
201 Then, run it:
202 
203  $ sudo ./blink.py
204 
205 You can get commutative HIGH and LOW from GPIO PG9, if you've run it on [IBOX](http://imall.iteadstudio.com/ibox.html/)
206 or other platforms based on Allwinner A10/A20 SoC.
207 
208 ## To use SDK in C program
209 
210 Create a new file named "led.c" which reads as follow:
211 
212  #include <itead.h>
213  #include <stdio.h>
214 
215  #define STATUS_LED1 PG9
216 
217  void setup(void)
218  {
219  printf("setup begin\n");
220  pinMode(STATUS_LED1,OUTPUT);
221  }
222  void loop(void)
223  {
224  digitalWrite(STATUS_LED1,HIGH);
225  delay(1000);
226  digitalWrite(STATUS_LED1,LOW);
227  delay(1000);
228  }
229  void main(int argc, char **argv)
230  {
231  setup();
232  while(1)
233  {
234  loop();
235  }
236  }
237 
238 Then run iteadcompile in the terminal
239 
240  $ iteadcompile LED led.c
241 
242 You can run the LED program after compilation:
243 
244  $ sudo ./LED

316 ### Note: How to understand debug information from compiler
317 
318 You DO NOT include the itead.h explicitly in your source file with suffix .ino.
319 iteadcompile will take care of everything. That means you need to know the method
320 you should use for locating your .ino sketch bugs. Actually,if you have a sketch
321 named "Hello.ino" with a bug at level ERROR in line 25, you will get some ERROR
322 informations telling the ERROR bug in "Hello.cpp" with line 26(25+1) when
323 command "iteadcompile Hello Hello.ino" is executed. Why? iteadcompile insert one
324 line like `include "itead.h"` in the head of your Hello.ino, change the suffix
325 .ino to .cpp and compile(using g++) Hello.cpp to generate Hello which is
326 executable binary file. When you encounter other bugs in your sketch,debug
327 skills on C++ program will also helpful for solving your problems.
